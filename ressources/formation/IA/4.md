# Introducción

Cuando decimos que una IA "aprende", muchas personas imaginan algo parecido a un niño aprendiendo a hablar.

No es exactamente así.

Una IA no entiende el mundo como nosotros.

No tiene experiencias, emociones ni sentido común.

Lo que hace es encontrar **patrones matemáticos** dentro de enormes cantidades de datos.

La palabra clave de este capítulo es:

> **Patrón**

Toda la inteligencia artificial moderna consiste en descubrir patrones.

---

# ¿Qué significa aprender?

Imagina que quieres distinguir perros y gatos.

Un humano puede hacerlo viendo unos pocos ejemplos.

Una IA necesita miles o millones.

Veamos el proceso.

```
Fotografía

↓

Extracción de características

↓

Comparación

↓

Ajuste del modelo

↓

Nueva fotografía

↓

Mejor decisión
```

Cada vez que comete un error, modifica ligeramente su forma de decidir.

Después de repetir este proceso millones de veces, termina aprendiendo.

---

# Un ejemplo sencillo

Supongamos que mostramos estas imágenes.

```
🐱 Gato

🐱 Gato

🐱 Gato

🐶 Perro

🐶 Perro

🐶 Perro
```

La IA no sabe qué es un gato.

Tampoco sabe qué es un perro.

Solo recibe:

Imagen → Etiqueta

```
Imagen001.jpg → gato

Imagen002.jpg → gato

Imagen003.jpg → perro
```

Al principio responde al azar.

Después de miles de ejemplos comienza a detectar diferencias.

No porque "entienda" qué es un gato.

Sino porque encuentra patrones estadísticos.

# Dataset

Un conjunto organizado de datos recibe un nombre muy importante.

## Dataset

Un dataset es simplemente una colección de ejemplos.

Ejemplo.

```
dataset/

    gatos/

        gato1.jpg

        gato2.jpg

    perros/

        perro1.jpg

        perro2.jpg
```

Los datasets pueden contener millones de archivos.

---

# El entrenamiento

El entrenamiento es el proceso mediante el cual una IA mejora poco a poco.

Podemos representarlo así.

```
Datos

↓

Modelo inicial

↓

Predicción

↓

Error

↓

Corrección

↓

Nueva predicción

↓

Nuevo error

↓

Corrección

↓

...

↓

Modelo entrenado
```

Este ciclo puede repetirse miles de millones de veces.

---

# ¿Qué es el error?

Imagina esta situación.

Respuesta correcta:

```
Gato
```

Respuesta de la IA.

```
Perro
```

Ha cometido un error.

Ese error debe medirse.

Cuanto mayor sea el error...

Mayor será la corrección.

La IA intenta reducir el error continuamente.

---

# Función de pérdida

Aquí aparece uno de los conceptos más importantes del Machine Learning.

## Loss Function

También llamada:

Función de pérdida.

Su misión es medir cuánto se ha equivocado el modelo.

Podemos imaginarla como un profesor corrigiendo un examen.

```
Alumno

↓

Examen

↓

Profesor

↓

Nota
```

La IA funciona de forma parecida.

```
Modelo

↓

Predicción

↓

Función de pérdida

↓

Error
```

El objetivo del entrenamiento consiste en hacer que ese error sea cada vez menor.

---

# Optimización

Una vez calculado el error...

Hay que corregir el modelo.

A este proceso lo llamamos optimización.

El algoritmo más famoso recibe el nombre de:

Gradient Descent

No te preocupes.

Lo estudiaremos en detalle más adelante.

Por ahora basta con saber que intenta mover el modelo en la dirección que reduce el error.

---

# Una analogía

Imagina que estás en una montaña completamente cubierta por niebla.

No puedes ver el valle.

Solo sabes si el siguiente paso sube o baja.

¿Qué harías?

Dar pequeños pasos hacia abajo.

Eso hace exactamente Gradient Descent.

Busca el punto donde el error es mínimo.

```
       /\

      /  \

     /    \

    /      \

   /        \

__/__________\____

       ↓

 Menor error
```

# Las épocas (Epochs)

Entrenar una IA significa recorrer el dataset muchas veces.

Cada recorrido completo recibe el nombre de:

Epoch

Ejemplo.

Dataset:

1000 imágenes.

```
Primera vuelta

↓

Epoch 1

↓

Segunda vuelta

↓

Epoch 2

↓

Tercera vuelta

↓

Epoch 3
```

Algunos modelos necesitan cientos de épocas.

Otros solo unas pocas.

---

# Batch

Imagina un millón de imágenes.

No sería eficiente procesarlas todas a la vez.

Por eso se dividen en pequeños grupos.

Cada grupo se llama:

Batch

Ejemplo.

```
1.000 imágenes

↓

100 batches

↓

10 imágenes por batch
```

Esto permite entrenar modelos enormes incluso con memoria limitada.

---

# El modelo

Cuando termina el entrenamiento obtenemos un modelo.

Podemos imaginarlo como un enorme conjunto de números.

No contiene reglas escritas por humanos.

Contiene millones o incluso billones de parámetros.

Ejemplo.

```
Entrenamiento

↓

Modelo IA

↓

Guardar

↓

Utilizar cuando sea necesario
```

---

# Inferencia

Una vez entrenado el modelo...

Ya no necesita aprender.

Solo debe responder.

Eso se llama:

Inferencia.

Cuando escribes:

```
¿Qué es un agujero negro?
```

ChatGPT no vuelve a entrenarse.

Simplemente utiliza el modelo ya aprendido.

---

# Entrenamiento vs Inferencia

| Entrenamiento                 | Inferencia           |
| ----------------------------- | -------------------- |
| Aprende                       | Utiliza lo aprendido |
| Muy lento                     | Muy rápido           |
| Consume mucha energía         | Consume poca         |
| Necesita millones de ejemplos | Solo una consulta    |

---

# Ejemplo completo

```
1. Reunir datos

↓

2. Limpiar datos

↓

3. Crear dataset

↓

4. Elegir modelo

↓

5. Entrenar

↓

6. Medir errores

↓

7. Mejorar parámetros

↓

8. Guardar modelo

↓

9. Utilizar modelo
```

Este ciclo es prácticamente igual para cualquier sistema moderno de IA.

# Código de ejemplo en Python

```python
# Ejemplo conceptual

imagenes = cargar_dataset()

modelo = crear_modelo()

for epoch in range(10):

    for imagen in imagenes:

        prediccion = modelo.predecir(imagen)

        error = calcular_error(prediccion)

        modelo.actualizar(error)

guardar(modelo)
```

Este código no funciona realmente, pero representa el proceso general de entrenamiento.

# Resumen

En este capítulo has aprendido que:

- La IA aprende encontrando patrones.
- Los datos son la base del aprendizaje.
- Un dataset es una colección organizada de ejemplos.
- El entrenamiento consiste en reducir continuamente el error.
- La función de pérdida mide ese error.
- La optimización ajusta el modelo para mejorar.
- El resultado del entrenamiento es un modelo.
- Cuando usamos ese modelo hablamos de inferencia.
